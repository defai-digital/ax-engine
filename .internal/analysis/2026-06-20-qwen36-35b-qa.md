## Bug Fixed

The server preset resolver for `qwen3.6-35b` did not accept the real 35B MoE `config.json` model type, `qwen3_5_moe`. As a result, `--resolve-model-artifacts hf-cache` rejected valid local Qwen 3.6 35B AX snapshots even though explicit `--mlx-model-artifacts-dir` launches worked.

Fix:

- Added `qwen3_5_moe` to the `qwen3.6-35b` server preset model-type allowlist.
- Kept the dense 27B preset on `qwen3_5`.
- Added regression coverage for 35B preset defaults and HF cache resolution against `qwen3_5_moe`.
