# Gemma 4 26B QA - 2026-06-20

## Scope

Runtime QA for the locally cached Gemma 4 26B variants:

- `mlx-community/gemma-4-26b-a4b-it-4bit`
- `ax-local/gemma-4-26b-a4b-it-assistant-mtp`
- `mlx-community/gemma-4-26b-a4b-it-6bit`

All live model probes used the native MLX server on Apple Metal with
`--preset gemma4-26b`, `--mlx`, and `--deterministic`.

## Results

The 4-bit OpenAI chat path passed coding, JSON, formatting, streaming, simple
arithmetic, and determinism probes. The exploratory report is
`/tmp/ax-gemma4-26b-qa.json`; the final classified summary is
`/tmp/ax-gemma4-26b-qa-final.json`.

The assistant-MTP package passed coding, JSON, and streaming probes. The package
contains `ax_gemma4_assistant_mtp.json` with schema
`ax.gemma4_assistant_mtp.v1`, `backend=gemma4_assistant`, and `max_depth=1`.
The saved report is `/tmp/ax-gemma4-26b-mtp-qa.json`.

The 6-bit package passed compact coding and JSON probes. The saved report is
`/tmp/ax-gemma4-26b-6bit-qa.json`.

## Findings

No confirmed runtime or server bug was found in the tested Gemma 4 26B paths.

One 4-bit arithmetic prompt returned `410` for `(17 * 23) + 48 - 19`, while a
simpler `14 + 28` arithmetic probe returned `42`. This was a deterministic
model-answer miss rather than a transport, decoding, or server failure.

The native `/v1/generate` text probe returned HTTP 400 because the MLX preview
native route requires pre-tokenized `input_tokens`; text input is supported
through the OpenAI chat/completions route. This matched the current route
contract and was not treated as a bug.
