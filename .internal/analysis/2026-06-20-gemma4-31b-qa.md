# Gemma 4 31B QA - 2026-06-20

## Scope

Runtime QA for the locally cached Gemma 4 31B variants:

- `mlx-community/gemma-4-31b-it-4bit`
- `ax-local/gemma-4-31b-it-assistant-mtp`
- `mlx-community/gemma-4-31b-it-6bit`

All live model probes used the native MLX server on Apple Metal with
`--preset gemma4-31b`, `--mlx`, and `--deterministic`.

## Results

The 4-bit OpenAI chat path passed Python coding, JavaScript coding, JSON,
formatting, streaming, arithmetic, native-route contract, and determinism
probes. The exploratory report is `/tmp/ax-gemma4-31b-4bit-qa.json`; the final
classified summary is `/tmp/ax-gemma4-31b-qa-final.json`.

The assistant-MTP package passed coding, JSON, and streaming probes. The package
contains `ax_gemma4_assistant_mtp.json` with schema
`ax.gemma4_assistant_mtp.v1`, `backend=gemma4_assistant`, and `max_depth=1`.
The saved report is `/tmp/ax-gemma4-31b-mtp-qa.json`.

The 6-bit package passed compact coding and JSON probes. The saved report is
`/tmp/ax-gemma4-31b-6bit-qa.json`.

A follow-up coding-regression pass was run after a user report that coding tasks
failed. The 4-bit path passed executable Python and JavaScript checks,
streaming SQL validation, strict JSON parsing, and deterministic repeat checks.
The assistant-MTP path passed executable Python, executable JavaScript, and
streaming SQL validation. The final coding summary is
`/tmp/ax-gemma4-31b-coding-qa-final.json`; source reports are
`/tmp/ax-gemma4-31b-coding-4bit-qa.json` and
`/tmp/ax-gemma4-31b-coding-mtp-qa.json`.

## Findings

No confirmed runtime or server bug was found in the tested Gemma 4 31B paths.
No confirmed coding-task bug was found in the stricter executable follow-up
pass.

The 4-bit `simple_arithmetic` row in the exploratory report is a QA-harness
false positive: the model returned exactly `42` for a prompt that requested only
the integer, but the generic harness length check marked the two-character
answer as too short.

The native `/v1/generate` text probe returned HTTP 400 because the MLX preview
native route requires pre-tokenized `input_tokens`; text input is supported
through the OpenAI chat/completions route. This matched the current route
contract and was not treated as a bug.
