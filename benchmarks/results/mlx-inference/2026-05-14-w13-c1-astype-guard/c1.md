# W1.3 C1 — astype dtype guard (2026-05-14)

PRD: `MLX-DECODE-SUBMIT-COST-PRD.md` W1.3
Audit: `MLX-DECODE-W13-CAST-RESHAPE-AUDIT.md` C1

## Change

Two `astype(_, Bfloat16)` call sites in the decode-1-token forward path
were unconditional but no-op when the input is already bf16. Added a
dtype guard:

- `crates/ax-engine-mlx/src/model.rs::forward_lazy_single_with_turboquant_context` L1980
- `crates/ax-engine-mlx/src/model.rs::compute_per_layer_inputs_arr` L2068

```rust
let embed_out = if embed_out.dtype() != dtype {
    astype(&embed_out, dtype, None)
} else {
    embed_out
};
```

## A/B (M5 Max, gemma-4-e2b-it-4bit, 128/128, 3 reps)

| Metric | Baseline | C1 | Δ |
|---|---:|---:|---:|
| decode_tok_s (median) | 185.41 | 185.23 | -0.10% |
| direct_pipeline_wall_us / step | 5,347.40 | 5,339.66 | -7.74 µs |
| direct_pipeline_async_eval_wall_us / step | 4,911.57 | 4,917.72 | +6.15 µs |
| **direct_pipeline_op_count / step** | **1,805** | **1,803** | **-2** |

Two ops removed per decode step (one per call site), confirming both
guards fired on the Gemma 4 4-bit hot path. Throughput delta is within
single-rep noise; no regression.

Cross-family safety: the guard is dtype-conditional, so non-bf16 model
families continue to receive the explicit cast unchanged.
